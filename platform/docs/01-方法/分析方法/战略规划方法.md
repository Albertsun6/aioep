# 战略规划方法

> 状态: **已细化** | 版本: v0.2.0 | 创建: 2026-02-16 | 上次验证: 未验证

## 定位

指导如何完成项目战略规划——从业务现状到可执行的战略目标和范围优先级。对应 TOGAF Phase A（架构愿景）。

## 前置条件

- 方法论体系已确认（AI-EADM）
- 项目负责人可参与（提供业务信息）
- 战略规划模板已准备（[02-项目/战略规划.md](../../02-项目/战略规划.md)）

## 步骤

### Step 1: 企业概况收集

**做什么**：了解企业基本面，建立业务上下文。

**AI 引导问题**：
1. 公司是做什么的？（行业、主要产品/服务）
2. 公司规模？（人数、年营收量级）
3. 组织架构？（有哪些部门，各自职责）
4. 客户群体？（B2B / B2C / 两者都有）
5. 业务地域范围？（本地 / 区域 / 全国 / 跨境）

**输出**：战略规划文档第 1 节"企业概况"

### Step 2: 业务愿景定义

**做什么**：明确公司未来 1~3 年的目标状态。

**AI 引导问题**：
1. 公司 1~3 年后想达到什么状态？
2. 有没有明确的增长目标？（营收 / 市场份额 / 客户数）
3. 有没有正在推进的业务变革？（新产品线 / 新市场 / 数字化转型）

**输出**：战略规划文档第 2 节"业务愿景"

### Step 3: 当前痛点分析

**做什么**：梳理当前最大的业务问题，按痛感排序。

**AI 引导问题**：
1. 目前最头疼的 3 个业务问题是什么？
2. 这些问题影响哪些部门/流程？
3. 目前怎么解决的？（Excel / 手工 / 其他系统）
4. 大约造成多大损失？（时间浪费 / 错误率 / 客户投诉）

**分析方法**：将痛点按"影响范围 x 痛感程度"排序，形成优先级。

**输出**：战略规划文档第 3 节"当前痛点"

### Step 4: 战略目标制定

**做什么**：将愿景转化为可衡量的目标（SMART 原则）。

**SMART 检查**：
- **S**pecific（具体）：目标是否明确？
- **M**easurable（可衡量）：有没有数字指标？
- **A**chievable（可实现）：现有资源能否达成？
- **R**elevant（相关）：是否与业务愿景一致？
- **T**ime-bound（有时限）：什么时候完成？

**AI 辅助方法**：
1. 人类描述期望 → AI 用 SMART 模板结构化
2. AI 提出反问帮助细化（"这个目标怎么衡量？""多长时间达成？"）

**输出**：战略规划文档第 4 节"战略目标"（含 KPI 和目标值）

### Step 5: 范围优先级排定

**做什么**：决定 ERP 覆盖哪些业务领域，哪些先做。

**排序矩阵**：

```
          高战略价值
              │
    P1 先做   │   P0 最先做
              │
低痛感 ───────┼─────── 高痛感
              │
    P3 以后做  │   P2 次优先
              │
          低战略价值
```

- **P0**：高战略价值 + 高痛感 → 第一优先
- **P1**：高战略价值 + 低痛感 → 第二优先
- **P2**：低战略价值 + 高痛感 → 第三优先
- **P3**：低战略价值 + 低痛感 → 放入后续版本

**输出**：战略规划文档第 5 节"范围与优先级"

### Step 6: 现有系统和数据盘点

**做什么**：了解现有工具和需要迁移的数据。

**AI 引导问题**：
1. 目前各业务领域用什么工具？（Excel / 纸质 / 其他系统）
2. 有多少历史数据需要迁移？
3. 数据质量如何？（干净 / 有重复 / 格式混乱）

**输出**：战略规划文档第 6 节"现有系统与数据"

### Step 7: 成功标准定义

**做什么**：定义"项目成功"的判断标准。

**方法**：从 Step 4 的战略目标反推——每个目标对应一个可观测的成功标准。

**格式**：`[动作] + [对象] + [量化指标] + [时间]`

示例：
- "采购流程处理时间从 3 天缩短到 1 天，上线后 1 个月内达成"
- "库存准确率从 85% 提升到 98%，上线后 2 个月内达成"

**输出**：战略规划文档第 7 节"成功标准"

### Step 8: 约束和风险识别

**做什么**：识别项目的硬约束和潜在风险。

**常见约束**：预算 / 时间 / 人员 / 技术能力 / 法规合规

**常见风险**：需求变更 / 数据质量 / 人员抵触 / 技术不成熟

**输出**：战略规划文档第 8 节"约束与风险"

### Step 9: 变革管理策略

**做什么**：规划如何推动组织接受新系统。

**AI 引导问题**：
1. 上线策略：先试点再推广，还是一步到位？
2. 谁是关键推动者？（哪个领导支持？）
3. 预期最大的阻力在哪里？
4. 培训怎么安排？

**输出**：战略规划文档第 9 节"变革管理策略"

## 检查清单

- [ ] 企业概况完整（行业/规模/架构/产品/客户）
- [ ] 愿景清晰可传达（一段话能说清方向）
- [ ] 痛点已排序（至少 3 个，有优先级）
- [ ] 每个战略目标符合 SMART（有 KPI 和目标值）
- [ ] 范围优先级已用矩阵排定（P0~P3）
- [ ] 现有系统和数据已盘点
- [ ] 成功标准可量化可观测
- [ ] 约束和风险已识别
- [ ] 变革管理策略已规划

## ArchiMate 建模对照

每个步骤同步在 Archi 中建模（Motivation + Strategy 层）：

| 步骤 | ArchiMate 元素 | 层 |
|------|-------------|---|
| Step 1 企业概况 | Stakeholder + Business Actor | Motivation |
| Step 2 愿景 | Goal（顶层） | Motivation |
| Step 3 痛点 | Driver + Assessment | Motivation |
| Step 4 目标 | Goal（细分）+ Outcome | Motivation |
| Step 5 优先级 | Capability + Course of Action | Strategy |
| Step 6 现有系统 | Resource（现有资源） | Strategy |
| Step 7 成功标准 | Outcome + Value | Motivation |
| Step 8 约束 | Constraint + Assessment | Motivation |
| Step 9 变革 | Course of Action（变革策略） | Strategy |
