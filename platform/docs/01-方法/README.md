# AI-EADM 实践方法

> 状态: **已确认** | 版本: v0.2.0 | 创建: 2026-02-15 | 更新: 2026-02-16

## 定位

**方法论层定义"用什么"，方法层定义"怎么做"。**

本层包含 AI-EADM 方法论各框架的具体执行方法——步骤、模板、操作规范、工具使用指南。按活动阶段组织，便于 AI 和人类在特定阶段快速定位所需方法。

每个方法有两个可执行翻译：
- **SOP**：人类侧执行标准（审批流、角色、异常处理）
- **Skills**：AI 侧执行指令（Prompt、约束、输出格式）

```
00-方法论（用什么）        01-方法（怎么做）              02-项目（实际产出）
──────────────           ──────────────              ──────────────
"用 TOGAF 做架构分析"     "Gap-Fit 的具体步骤和模板"    "我们的 Gap-Fit 报告"
"用 BPMN 建模流程"        "draw.io 画 BPMN 的规范"     "采购流程 BPMN 图"
"用 Scrum 跑迭代"         "Sprint 的执行流程"          "Sprint 1 Review 记录"
"用 TDD 写测试"           "Odoo 测试怎么写"            "erp_base 测试代码"
```

## MECE 检查

| 维度 | 检查 |
|------|------|
| ME（相互独立） | 分析/开发/交付 三个目录对应三个不同阶段，不重叠 |
| CE（完全穷尽） | Phase 1~3 的所有"怎么做"都有归属；Phase 0 的方法已在方法论层完成 |
