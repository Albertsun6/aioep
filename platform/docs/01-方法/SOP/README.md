# SOP（标准操作程序）

> 状态: **框架已建立** | 版本: v0.1.0 | 创建: 2026-02-22

## 定位

SOP 是方法 (Method) 的**人类侧可执行翻译**，与 Skills（AI 侧可执行翻译）互补。

```
方法 (Method) ─── "做什么 + 步骤"
  ├── SOP    ─── "人类怎么执行"（审批流、角色、异常处理）
  └── Skills ─── "AI 怎么执行"（Prompt、约束、输出格式）
```

## SOP vs Skills vs 方法

| 维度 | 方法 | SOP | Skills |
|------|------|-----|--------|
| 回答 | 做什么、分几步 | 人类怎么做、谁审批 | AI 怎么做、输出什么 |
| 受众 | 所有人 | 人类执行者 | AI Agent |
| 形式 | 操作手册 | 流程图 + 检查表 + 角色矩阵 | System Prompt |
| 变更 | 方法改进时 | 组织调整时 | Prompt 调优时 |

## SOP 编写规范

每个 SOP 文档应包含以下章节：

1. **触发条件** — 何时启动此 SOP
2. **角色与权限** — 谁参与、各自职责
3. **操作流程** — 步骤 + 判断分支 + 审批节点
4. **输入/输出** — 每步的输入物和交付物
5. **异常处理** — 超时、退回、升级规则
6. **检查清单** — 完成标准
7. **与 Skills 的协作点** — 哪些步骤由 AI 执行

## 命名规范

`SOP-{业务域}-{流程名}.md`，例如：
- `SOP-战略-建模审阅.md`
- `SOP-项目-启动评审.md`
- `SOP-开发-Sprint计划.md`
