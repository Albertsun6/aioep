# 需求建模方法

> 状态: **纲要** | 版本: v0.1.0 | 创建: 2026-02-16 | 上次验证: 未验证

## 定位

指导如何将业务需求转化为结构化模型——BPMN 流程图、ArchiMate 架构图、UML 数据模型、RBAC 权限矩阵、BDD 行为规格。

## 前置条件

- 战略规划已完成
- Gap-Fit 分析已完成（知道哪些需要建模）
- Archi 和 draw.io 工具已安装
- 建模标准已确认（[03-建模体系](../../00-方法论/03-建模体系/README.md)）

## 步骤

1. 企业架构建模（ArchiMate → Archi 工具）
   - 业务层 → 应用层 → 技术层三层全景
2. 核心业务流程建模（BPMN → draw.io）
   - 采购/销售/库存/财务/审批 至少 5 个流程
3. 数据模型设计（UML Class → draw.io）
   - 核心模型及关系（基于 DDD 限界上下文划分）
4. 权限角色设计（RBAC 矩阵 → Markdown 表格）
5. 关键场景行为规格（BDD → Given/When/Then）

> 各步骤的工具操作规范待细化。当内容增多时，可能拆分为子方法（bpmn操作规范.md / archimate操作规范.md 等）。

## 模板

- ArchiMate: 详见 [archimate.md](../../00-方法论/03-建模体系/archimate.md) L1 元素清单
- BPMN: 详见 [bpmn.md](../../00-方法论/03-建模体系/bpmn.md) L1 元素清单 + 流程清单
- BDD: 详见 [04-设计模式](../../00-方法论/04-设计模式与开发实践/README.md) Given/When/Then 格式
