# 09 - 工件（Artifacts）

> 状态: **纲要** | 版本: v0.1.0 | 创建: 2026-02-15

## 定位

定义项目中**每个阶段产出什么**。工件是方法论的有形产出——文档、模型、代码、测试、配置。明确每个工件的名称、格式、责任人、验收标准和存储位置。

## 工件总览

### 按类型分类

| 类型 | 说明 | 格式 | 存储位置 |
|------|------|------|---------|
| 文档 | 需求、设计、手册等文字文档 | Markdown (.md) | docs/ |
| 模型 | ArchiMate/BPMN/UML/C4 建模文件 | .archimate / .drawio | models/ |
| 模型导出 | 模型图片，供文档引用 | PNG / SVG | models/exports/ |
| 代码 | Odoo 模块源代码 | Python / XML / JS | custom_addons/ |
| 测试 | 单元测试、集成测试 | Python (pytest) | custom_addons/*/tests/ |
| 配置 | Docker、CI/CD、Odoo 配置 | YAML / conf / JSON | config/ / docker/ |
| 数据 | 初始数据、迁移脚本 | XML / Python | custom_addons/*/data/ |
| 记录 | 对话记录、变更日志 | Markdown | docs/Chatlog.md / CHANGELOG.md |

## 工件命名规范

```
文档：   {主题}.md                     如 项目章程.md
模型：   {模块名}-{图类型}.drawio       如 purchase-process.drawio
导出：   {模块名}-{图类型}.png          如 purchase-process.png
代码：   erp_{模块名}/                  如 erp_sale/
测试：   test_{功能名}.py               如 test_purchase_approval.py
```
