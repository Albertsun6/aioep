# AI-EADM 方法论总纲

> **AI-EADM**: AI Enterprise Application Development Methodology（AI 企业应用开发方法论）
>
> 状态: **核心已确认** | 版本: v0.5.0 | 创建: 2026-02-15 | 更新: 2026-02-15

AI-EADM 是一套面向 AI 驱动的企业应用系统开发的方法论体系。本文档是其最高层级文档（"项目宪法"），所有项目活动、文档、代码和决策必须遵循此方法论。

AI-EADM 不仅适用于 ERP，也适用于 CRM、SCM、OA 等任何企业级应用系统的开发。

> **核心目标声明**：本项目的首要目标是**构建和完善 AI-EADM 方法论体系本身**。ERP 项目是方法论的第一个应用实例，用于验证和迭代方法论。方法论的结构完善优先于项目交付进度。

## 方法论体系全景

```
AI-EADM 方法论体系
│
├── 01-核心原则                  ← 价值观和底线，约束一切

│   ├── DevOps   → 持续交付
│   ├── ITIL     → 服务管理
│   └── Lean     → 效率优化
│

│   ├── ArchiMate → 企业架构建模（Archi 工具）

│
├── 04-设计模式与开发实践          ← 怎么写出好代码
│   ├── TDD      → 测试驱动开发
│   ├── BDD      → 行为驱动开发
│   ├── DDD      → 领域驱动设计
│   ├── EDA      → 事件驱动架构
│   └── Odoo Patterns → Odoo 专用设计模式
│
├── 05-质量框架                  ← 怎么保证质量
│   五层防线 + 质量门禁 + 度量指标
│

│   RACI 矩阵 + 评审机制 + 变更管理
│
├── 07-AI 协作协议               ← 怎么与 AI 协作
│   输出规范 + 代码注释 + Chatlog 规范

└── 09-工件                      ← 每个阶段产出什么
    文档 / 模型 / 代码 / 测试 / 配置（按 Phase 分类）
```
