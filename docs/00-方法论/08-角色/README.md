# 08 - 角色（Roles）

> 状态: **纲要** | 版本: v0.1.0 | 创建: 2026-02-15

## 定位

定义项目中**谁来做**。明确每个角色的职责、权限、参与阶段和协作关系。每个角色在每个阶段做什么、不做什么、与谁交接。

## 角色清单

| 角色 | 英文 | 定义 | 参与阶段 |
|------|------|------|---------|
| 项目负责人 | Project Owner | 业务决策、需求优先级、验收审批 | 全程 |
| 开发者（人类） | Developer | 架构决策、代码审查、AI 输出验证、部署 | 全程 |
| AI 编程助手 | AI Assistant | 代码生成、文档生成、测试生成、问题分析 | 全程 |
| 业务顾问 | Business Consultant | 需求输入、业务流程确认、UAT 测试 | Phase 1 / 4 |
| 财务/审计 | Finance/Audit | 财务规则确认、数据流审查、合规审计 | Phase 1 / 3 / 4 |
| 管理层 | Management | 项目审批、资源分配、战略决策 | 里程碑节点 |
| 未来开发者 | Future Developer | 接手维护、功能迭代（暂未参与） | Phase 5+ |

## 人类与 AI 的分工原则

```
人类负责（不可委托给 AI）：
├── 业务判断和需求优先级
├── 架构级别的技术决策
├── 安全策略和合规决策
├── 代码最终审查和合入
├── UAT 验收和上线审批
└── 关键风险的研判

AI 负责（人类审查后生效）：
├── 代码生成和重构
├── 测试用例生成
├── 文档生成和更新
├── 模型草稿（Mermaid）
├── 问题分析和方案建议
├── 日志分析和性能建议
└── Chatlog 记录
```
