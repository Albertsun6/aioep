# Phase 1: 需求分析与设计

## 上下文获取

1. 读取项目上下文：`.cursor/rules/project-context.md`
2. 确认当前在 Phase 1 的哪个活动、哪个步骤
3. 如果 AIOEP 仓库在工作区内，读取方法论和方法文件

## Phase 1 活动导航

Phase 1 包含以下活动，按顺序执行：

### 活动 1: 战略规划
- **方法文件**: AIOEP 仓库 `docs/01-方法/分析方法/战略规划方法.md`
- **模板**: AIOEP 仓库 `docs/01-方法/分析方法/模板-战略规划.md`
- **输出**: `docs/02-项目/战略规划.md`
- **检查清单**: 方法文件末尾的检查清单

### 活动 2: Gap-Fit 分析
- **方法文件**: AIOEP 仓库 `docs/01-方法/分析方法/gap-fit方法.md`
- **输出**: `docs/02-项目/gap-fit报告.md`
- **步骤**: 读取方法文件中的 6 个步骤，按步骤引导执行

### 活动 3: 需求建模
- **方法文件**: AIOEP 仓库 `docs/01-方法/分析方法/需求建模方法.md`
- **输出**: BPMN 流程图 + 用例文档
- **工具**: draw.io（BPMN）、Markdown（用例描述）

### 活动 4: 系统架构设计细化
- **输入**: Gap-Fit 报告 + 需求模型
- **输出**: `docs/02-项目/系统架构设计.md`（从草稿升级为已确认）

### 活动 5: Phase 1 评审
- **检查**: 方法论 `docs/00-方法论/05-质量框架/README.md` 中的 Phase 1 Exit 条件
- **输出**: Phase 1 评审记录

## 执行规则

1. **按顺序**：严格按活动 1→2→3→4→5 执行，不跳跃
2. **读方法文件**：每个活动开始前，先读取对应的方法文件了解步骤
3. **跟踪进度**：每完成一个步骤，提醒用户更新 project-context.md
4. **引用模板**：输出文档使用方法文件中指定的模板格式
5. **自检**：每个活动完成后，对照方法文件的检查清单自检

## 当前状态判断

读取 project-context.md 后：
- 如果"当前活动"是战略规划 → 引导活动 1
- 如果"当前活动"是 Gap-Fit → 引导活动 2
- 如果"当前活动"是需求建模 → 引导活动 3
- 以此类推

## 完成后

1. 更新 `project-context.md`：当前阶段改为 Phase 2，当前活动改为 Sprint 0
2. 更新 `docs/Chatlog.md`：记录 Phase 1 完成
3. 提醒用户：Phase 1 评审通过后，可进入 Phase 2（迭代开发）
